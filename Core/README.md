
در فایل هسته تابع find_best_route مسئول ساخت مسیریابی و زمان بندی و راهنمایی کاربر است.
در این فایل گراف ها از قبل برای تسریح پاسخ دهی این فایل ساخته و در حافظه قرار میگیرن تا در هر بار اجرای تابع گراف ها از ابتدا ساخته نشن.

این تابع 2 ورودی میگیرد. اولی نام ایستگاه مبدا و نام ایستگاه مقصد. با هر ورودی تابع مشابه ترین نام ایستگاه را با ورودی ها جای گذاری میکند تا مشکلات نگراشی برطرف بشن.

اگر ایستگاه مبدا و مقصد یکی باشند خطا دریافت میشود و اگر ورودی ها جوری باشن که شباهت دوری هم نشه بهشون داد با نام ایستگاه ها باز هم خطا دریافت خواهد شد.

اگر ایستگاه مبدا و مقصد در یک خط باشند فقط گراف همان خط برای مسیریابی استفاده میشود و از گراف کامل که شامل تمامی ایستگاه ها و خطوط مترو هستش استفاده نخواهد شد.

اگر خط های ایستگاه مبدا و مقصد متفاوت باشند از گراف کامل استفاده خواهد شد.
